PART 4: MARK ENTRY FORM (index.html + form.js)
────────────────────────────────────────────────────────────────────────────────

The form is the data entry interface for teachers. It is NOT the report card.
It is a clean, functional form that feeds data into the report card renderer.

FORM SECTIONS:

  SECTION A — CLASS & STUDENT INFO
    - Class selector dropdown (III, IV, V, VI, VII, VIII, IX, X)
      → On change: reload subject rows + mark scheme dynamically
    - Student Name (text)
    - Roll Number (number)
    - Section (text, optional — e.g., "A")
    - Class Teacher Name (text)
    - Admission Number (text)
    - Date of Birth (date)
    - House (text — e.g., "Francis House")
    - Session (pre-filled: "2026-2027", read-only)

  SECTION B — HALF YEARLY MARKS TABLE
    Rendered dynamically based on selected class.
    Columns for standard scheme: Subject | IA(/10) | UT(/30) | Exam(/60) | Total
    Columns for senior scheme:   Subject | IA(/20) | Exam(/80) | Total
    - Each input enforces its max value on blur (show error if exceeded)
    - Total auto-calculates on any input change (read-only)
    - Aggregate rows auto-calculate (read-only, visually distinct)
    - Grand Total row auto-calculates (read-only, bold)
    - Percentage auto-calculates: (grandTotal / grandTotalMax) × 100
    - Attendance row: two inputs → [Present] / [Total Working Days]

  SECTION C — FINAL TERM MARKS TABLE
    Same structure as Section B.
    Consolidated column auto-calculates: HY Total + FT Total per subject

  SECTION D — CO-SCHOLASTIC GRADES
    For each co-scholastic subject (from class config):
      - Subject label
      - Half Yearly grade dropdown (A+, A, B+, B, C+, C, D)
      - Final Term grade dropdown (A+, A, B+, B, C+, C, D)

  SECTION E — REMARKS
    - Half Yearly Remark: textarea (class teacher types freely)
    - Final Term Remark: textarea (class teacher types freely, include
      promotion status e.g. "Promoted to Class IV (Four)")

  SECTION F — RANK
    - Half Yearly Rank: number input (manual entry in Phase 1)
    - Final Term Rank: number input (manual entry in Phase 1)
    - Total students in class: number input (for display as "Rank / Total")
    - Show a note: "Rank is eligible only if student passes all subjects
      (40+ in each). Dense ranking — no rank skipped for ties."

  SECTION G — RESULT
    Auto-determined:
    - If ALL countInTotal subjects score ≥ 40 in BOTH terms → PASS
    - If ANY countInTotal subject scores < 40 in either term → FAIL
    - Display auto-result, allow teacher to see it (read-only indicator)

FORM ACTIONS (buttons at bottom):
    [Preview Report Card]   → renders reportcard.html in a new tab/window
    [Print Report Card]     → triggers print of the report card (legal size)
    [Save to JSON]          → downloads current form data as .json file
    [Load from JSON]        → loads previously saved .json to repopulate form
    [Clear Form]            → resets all fields (with confirmation prompt)
    [Add to Class List]     → saves current student to session memory
                              for class marksheet generation
    [View Class Marksheet]  → opens marksheet.html populated with all
                              students added to session memory

PART 7: CALCULATION LOGIC (form.js)
────────────────────────────────────────────────────────────────────────────────

SUBJECT TOTAL:
  Standard: IA + UnitTest + Exam (capped at 100)
  Senior:   IA + Exam (capped at 100)
  Single:   Direct total input (e.g., Spelling)

AGGREGATE TOTAL:
  Auto-sum of component subject totals (read-only field)

GRAND TOTAL:
  Sum of all subject totals where countInTotal === true

PERCENTAGE:
  (grandTotal / grandTotalMax) × 100, rounded to 2 decimal places

GRADE (from percentage):
  ≥ 90  → O  (Outstanding)      [Only use if school wants O grade]
  80–89 → A+
  70–79 → A
  60–69 → B+
  50–59 → B
  40–49 → C
  33–39 → D
  < 33  → F (Fail)
  Pass mark: 40/100

  NOTE: Use the grading scale confirmed by the school. The sample design
  shows O/A+/A/B+/B/C/D/F. Implement exactly this scale.

CONSOLIDATED (per subject):
  Term1 subject total + Term2 subject total = Consol. (out of 200)

OVERALL CONSOLIDATED TOTAL:
  Term1 Grand Total + Term2 Grand Total (out of grandTotalMax × 2)

RESULT:
  Check ALL countInTotal subjects in BOTH terms.
  If ANY subject total < 40 → FAIL
  If ALL subject totals ≥ 40 → PASS

RANK ELIGIBILITY:
  Student must PASS (all subjects ≥ 40) to receive a rank.
  Rank is manually entered in Phase 1.

ATTENDANCE PERCENTAGE:
  (daysPresent / totalWorkingDays) × 100

INPUT VALIDATION:
  - IA: 0 to max (10 or 20)
  - Unit Test: 0 to 30
  - Exam: 0 to max (60 or 80)
  - Show inline red error if value exceeds max or is negative
  - Do not allow form preview if required fields are empty

PART 8: DATA FLOW
────────────────────────────────────────────────────────────────────────────────

1. Teacher fills the form on index.html
2. On "Preview Report Card" click:
   a. Collect all form data into a JS object (studentData)
   b. Store in sessionStorage as JSON string
   c. Open reportcard.html in new tab
   d. reportcard.html reads sessionStorage, renders the full card
3. On "Print Report Card" click:
   a. Same as above but immediately calls window.print() after render
4. On "Save to JSON" click:
   a. Download studentData as [StudentName]_[Class]_[Session].json
5. On "Load from JSON" click:
   a. File picker opens
   b. Loaded JSON repopulates the form
6. On "Add to Class List" click:
   a. Push studentData into a classListArray in sessionStorage
   b. Show confirmation: "Student [Name] added. Total: [N] students."
7. On "View Class Marksheet" click:
   a. Open marksheet.html
   b. marksheet.html reads classListArray from sessionStorage
   c. Renders the consolidated table for all added students
   d. Sorts by Roll Number

PART 9: IMPORTANT IMPLEMENTATION NOTES
────────────────────────────────────────────────────────────────────────────────

1. DO NOT use any external CSS framework (no Bootstrap, no Tailwind).
   Write all CSS from scratch using the color palette defined in Part 2.

2. DO NOT use any JavaScript framework (no React, no Vue).
   Pure Vanilla JS only.

3. The report card HTML template in reportcard.html must look
   IDENTICAL on screen and when printed. Test both views.

4. Aggregate rows in the marks table must be visually distinct:
   slightly bolder font, a subtle gold-tinted background row,
   and "(auto)" or a formula indicator hidden from print.

5. The bar chart in the Consolidated Subject Performance section
   must be built using pure CSS (div with width % set by JS).
   Do NOT use any chart library.

6. Component subject rows (Geography, Civics, History etc.) that
   do NOT count toward Grand Total must be visually de-emphasized:
   smaller font, lighter text, slight indent on the subject name.

7. All MAX values must be displayed in column headers:
   e.g., "IA /10" not just "IA" — this matches the physical card format.

8. The Consolidated column in Term 2 table shows "/200" in the header
   (Term1 Total + Term2 Total per subject, max 200).

9. The overall consolidated grand total is out of (grandTotalMax × 2).
   E.g., Class X: 600 × 2 = 1200. Class III: 800 × 2 = 1600.

10. For Spelling (singleTotal subjects): show only one input column
    spanning IA+UT+Exam columns, labeled "Total" directly.
    No breakdown shown for these subjects.

11. When class changes in the form, ALL subject rows, calculations,
    and column headers must re-render dynamically. Do not reload the page.
    Use JS DOM manipulation to rebuild the table.

12. Session is pre-filled as "2026-2027" and is read-only in the form.

13. School name must auto-switch based on class:
    Classes III–VII, IX, X → "St. Francis De Sales School"
    Class VIII              → "St. Francis De Sales Sec. School"

14. Date of Issue field in the report card footer is blank (printed,
