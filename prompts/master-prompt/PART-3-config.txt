PART 3: CLASS CONFIGURATION (config.js)
────────────────────────────────────────────────────────────────────────────────

Build a CONFIG object in config.js. Each class entry defines:
  - schoolName       : string (differs for upper classes)
  - grandTotalMax    : number
  - markScheme       : "standard" | "senior"
      standard = IA(10) + UnitTest(30) + Exam(60) = 100
      senior   = IA(20) + Exam(80) = 100  [Class IX & X only]
  - subjects         : array of subject objects
  - coScholastic     : array of activity subject names
  - aggregates       : array of aggregate definitions

Subject object shape:
  {
    key        : "mathematics",          // unique key
    label      : "Mathematics",          // display name
    isAggregate: false,                  // true = auto-sum, no input
    components : [],                     // keys of subjects that feed this
    countInTotal: true,                  // false = component rows excluded
    singleTotal : false                  // true = Spelling-type, one input only
  }

Aggregate logic:
  - Component subjects (e.g., Geography, Civics, History) have
    countInTotal: false — they display but do NOT add to Grand Total
  - The aggregate row (e.g., S.Science G+C+H) has countInTotal: true
    and isAggregate: true — it IS added to Grand Total
  - English I and English II similarly feed into English (I+II) aggregate
